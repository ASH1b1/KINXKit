# 计划与实现文档索引

> 原则：先有文档，后有代码；测试先行，验证通过再进入下一阶段。

## 目录规范
- 存放位置：docs/2026.03.09-ZHANGplans/
- 命名规则：YYYY-MM-DD-<feature-name>.md（例如：2026-03-09-proxy-detector.md）
- 内容要求：
  - 目标与范围（明确“做什么/不做什么”）
  - 接口与数据结构（精确到文件路径与字段）
  - 架构与策略（关键决策与平台适配）
  - 测试计划（TDD 用例、期望输出、覆盖策略）
  - 执行步骤（按最小粒度分解，包含预期结果）
  - 验收标准（通过条件与日志/安全约束）

## 工作流
1. 撰写与提交计划文档（本目录）
2. 审核通过后，先写测试（确保失败）
3. 最小实现使测试通过
4. 扩充分支与异常测试，确保稳定
5. 代码与文档同步更新，进入下一项
